The two features listed above are as follow-


    If you fire either Get Request or Post Request then you take to a page where essentially you have to type in some data and click submit


    If the request made is GET, then you can clearly see the data input in the URL Bar as the data is passed as params inside the request header


    Else if the request is POST, and the data input will be passed inside the body of the request thus it is not shown on URL


By clicking on either of the button you are making a GET request without header params set
